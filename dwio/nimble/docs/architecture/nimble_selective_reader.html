1 ChunkedDecoder — Two Read Paths

ChunkedDecoder exposes multiple APIs — which path depends on the reader

Selective Reader (production)

SelectiveNimbleReader → SelectiveColumnReader

APIs used:
• readWithVisitor(visitor) — always, even without filter
• skip(n) — for seekTo

Without filter: visitor is AlwaysTrue — every row passes
With filter: visitor applies predicate inline during decode

Uses bulkScan with SIMD optimizations

Batch Reader (non-selective)

VeloxReader → FieldReader → Decoder

APIs used:
• materialize(n, buffer) — decode N values into buffer
• skip(n) — advance without decoding

No filter support: reads all rows, filters applied post-decode

Different code path — no visitor, no bulkScan
⚠
These are different read paths with different performance characteristics. The selective path's readWithVisitor uses SIMD-accelerated bulkScan that the batch path's materialize() does not. Benchmarks using the batch reader may not reflect production selective reader performance.

Advances position without decoding values.

3 Internal: encoding_ — Encoding Object

encoding_ is a unique_ptr<Encoding> on ChunkedDecoder. Virtual dispatch to concrete type. The encoding never sees the filter — it just decodes values and calls visitor.process(value).

EncodingFactory::create()

Recursive descent parser. Reads prefix, switches on EncodingType, recursively builds tree. Leaf encodings decompress eagerly in their constructors. Example: nullable int32 column.

4 Internal: encoding_ — readWithVisitor

1. Visitor creation

Visitor construction (in ColumnReader::read())
visitor = ColumnVisitor<T, Filter, ExtractToReader, isDense> // compile-time template
  .filter_ = scanSpec filter // e.g. col1 > 50
  .rows_ = row indices // [0..N] dense or [3,17,42,...] sparse
  .outputRows_, rawValues_ // references to column reader's buffers
decoder_.readWithVisitor(visitor) // pass to ChunkedDecoder → encoding

2. Dense vs Sparse

Dense (isDense = true)
rows = [N, N+1, N+2, ..., M] — full batch scan, no gaps check: rows.back() == rows.size() - 1 (0-based within batch)
first filter column always dense (rows from iota())
Sparse (isDense = false)
rows = [3, 17, 42, ...] — gaps between survivors
subsequent filter columns get sparse from previous filter

5 Encoding Decode — Fast & Slow Paths
encoding_->readWithVisitor(visitor)
virtual dispatch to concrete encoding type
bulkScan essence: decode first, filter second
Step 1 Decode Batch (X rows): decode all selected values into flat buffer
  dense → SIMD batch decode (big win)
  sparse → random access loop (marginal win)
Step 2 Filter Batch (X rows): processFixedWidthRun() — filter the buffer in one pass
  filter + scatter over contiguous buffer
  cache friendly, branch predictor friendly
readWithVisitorSlow: decode and filter interleaved
decode one → filter one → decode one → filter one → decode one → filter one...

Concrete examples

col1 (FixedBitWidth, dense, filter: > 50) — Fast path
packed data: [12, 73, 3, 99, 51] (already decompressed in constructor)

Step 1: bulkScan → SIMD decode all 5 values into buffer
  buffer = [12, 73, 3, 99, 51]

Step 2: processFixedWidthRun() → filter each
  12 → 12 > 50? NO → discard
  73 → 73 > 50? YES → outputRows_[0]=1, rawValues_[0]=73
  3  → 3 > 50?  NO → discard
  99 → 99 > 50? YES → outputRows_[1]=3, rawValues_[1]=99
  51 → 51 > 50? YES → outputRows_[2]=4, rawValues_[2]=51

Result: numValues_ = 3
col3 (Trivial, sparse rows=[1,3,4], filter: < 3.14) — Fast path
raw values: [1.1, 2.7, 8.5, 0.3, 5.1]

Step 1: bulkScan → random access selected rows
  values_[1] = 2.7, values_[3] = 0.3, values_[4] = 5.1

Step 2: filter each
  2.7 → 2.7 < 3.14? YES → outputRows_[0]=1, rawValues_[0]=2.7
  0.3 → 0.3 < 3.14? YES → outputRows_[1]=3, rawValues_[1]=0.3
  5.1 → 5.1 < 3.14? NO → discard

Result: numValues_ = 2
